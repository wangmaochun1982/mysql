Veritas 备份工具通常是收费的，其收费模式受多种因素影响，下面为你详细介绍：

#### 收费影响因素

- **功能模块**：Veritas 提供了多种不同功能的备份工具产品，像 NetBackup、Backup Exec 等，每个产品又具备不同的功能模块。例如，NetBackup 如果使用高级的数据管理、灾难恢复、云集成等功能模块，需要额外付费，功能越高级、越全面，收费越高。
- **许可类型**：有按物理服务器数量许可、按虚拟服务器数量许可、按存储容量许可等多种类型。比如按物理服务器数量许可，企业有更多的物理服务器需要使用 Veritas 备份工具，购买的许可数量就多，费用也会相应增加。
- **使用时长**：一般提供年度订阅许可和永久许可两种。年度订阅许可需要每年支付费用以持续使用软件及获得技术支持和软件更新；永久许可则是一次性支付费用获得软件使用权，但后续的技术支持和软件更新可能需要额外付费。

#### 价格举例

- 对于小型企业适用的 Veritas Backup Exec，如果是基础功能版本，每年的订阅费用可能在几百美元到几千美元不等，具体取决于许可数量和功能模块。
- 大型企业使用的 NetBackup，由于其功能复杂、可扩展性强，价格相对较高。如果是大型跨国企业，涉及大量服务器和数据存储，其购买许可和服务的费用可能每年会达到数十万美元甚至更高。

#### Veritas NetBackup 搭建教程

**一、系统要求**

1. **操作系统**：Windows Server 2016+/Linux RHEL 7+（根据版本匹配）
2. **硬件**：4核CPU、16GB内存、100GB磁盘空间（主服务器建议更高配置）
3. **依赖环境**：Java Runtime Environment (JRE)、浏览器（管理控制台访问）
4. **网络**：开放TCP端口1556（主服务器通信）、13724（介质服务器）

**二、安装前准备**

1. **获取安装包**

   从Veritas官网下载NetBackup安装镜像（需有效许可证）。

2. **创建专用账户**

   ```
   # Linux示例：创建nbuser账户
   sudo useradd -m nbuser
   sudo passwd nbuser
   ```

3. **配置防火墙**

   ```
   # 开放主服务器端口（Linux）
   sudo firewall-cmd --permanent --add-port=1556/tcp
   sudo firewall-cmd --reload
   ```

**三、主服务器安装（Linux示例）**

1. **挂载ISO/解压安装包**

   ```
   sudo mount -o loop nbu_install.iso /mnt
   cd /mnt/install
   ```

2. **运行安装程序**

   ```
   sudo ./install
   ```

   - 选择 **"NetBackup Master Server"** 组件。

3. **配置主服务器**

   - 输入主机名（需解析正确）。
   - 导入许可证文件（`.lic`格式）。
   - 设置存储单元默认路径（如`/usr/openv/netbackup`）。

4. **验证安装**

   ```
   sudo systemctl status nb服务
   # 访问管理控制台：https://<主服务器IP>:1556
   ```

**四、介质服务器安装（可选）**

1. 运行安装程序时选择 **"Media Server"**。

2. 指定主服务器IP地址完成关联。

3. 添加存储设备（磁带库/磁盘池）：

   ```
   /usr/openv/volmgr/bin/sgsetup  # 配置存储单元
   ```

**五、客户端安装**

1. **Windows客户端**

   - 运行`setup.exe`，选择 **"NetBackup Client"**。
   - 输入主服务器IP地址。

2. **Linux客户端**

   ```
   sudo ./install --client
   ```

**六、配置备份策略**

1. **登录管理控制台**\ 浏览器访问 `https://<主服务器IP>:1556`，使用`admin`账户登录。
2. **创建存储单元**
   - **策略和存储 > 存储单元 > 新建**
   - 类型选择 **"磁盘"** 或 **"磁带"**，指定路径/设备。
3. **新建备份策略**
   - **策略和存储 > 策略 > 新建**
   - 设置备份频率（如每日全备）、保留周期（如30天）。
   - 添加客户端并选择备份文件（如`/data`目录）。

**七、执行首次备份**

1. **手动启动任务**
   - 在策略页面右键选择 **"立即备份"**。
2. **监控任务状态**
   - 查看 **"活动监视器"** 确认任务完成，无错误提示。

**八、常见问题**

1. **客户端无法连接**
   - 检查主服务器防火墙规则、`bp.conf`配置文件中的IP地址。
2. **备份失败（状态6）**
   - 确认客户端服务运行：`systemctl status nbclient`。
   - 检查存储单元权限：`chmod -R 755 /backup_storage`。
3. **许可证过期**
   - 在 **"管理控制台 > 许可证"** 中更新许可证文件。

**九、注意事项**

- **定期维护**：清理过期备份、监控存储空间。
- **日志位置**：`/usr/openv/netbackup/logs/`（Linux）。
- **升级规划**：通过`./install`选择升级选项前备份配置。

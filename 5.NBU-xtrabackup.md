你看到的：--incremental-basedir = 全备目录（ekptest-bak-15）

而不是：--incremental-basedir = 上一次增量目录

说明：🔥 每一次增量都是基于“同一个全备”，而不是基于上一个增量


1️⃣ 链式增量（Cumulative / Incremental Chain）

每次基于“上一次备份”：

全备 → 增量1 → 增量2 → 增量3
        ↑        ↑
      基于全备  基于增量1

命令应该是：

# 第二次增量（正确链式写法）
--incremental-basedir=增量1目录

👉 特点：

恢复要一层一层 apply
链断就完蛋
占空间小
2️⃣ 差异增量（Differential）✅（你现在这个）

每次都基于“同一个全备”：

全备 → 增量1（基于全备）
     → 增量2（基于全备）
     → 增量3（基于全备）

给你一版“标准恢复脚本”（可直接用）

```bash
#!/bin/bash

FULL=/restore/full
INCR=/restore/incr_125
DATADIR=/data/mysql

# 1. prepare full
xtrabackup --prepare --apply-log-only --target-dir=$FULL

# 2. apply incr
xtrabackup --prepare --apply-log-only \
  --target-dir=$FULL \
  --incremental-dir=$INCR

# 3. finalize
xtrabackup --prepare --target-dir=$FULL

# 4. stop mysql
systemctl stop mysqld

# 5. clean datadir
rm -rf ${DATADIR}/*

# 6. copy back
xtrabackup --copy-back --target-dir=$FULL

# 7. fix permission
chown -R mysql:mysql $DATADIR

# 8. start mysql
systemctl start mysqld
```

### **💡 关键参数说明**

- `--incremental-basedir`

  : 这是增量备份的灵魂。

  - 如果是**第一次增量备份**，这里填你的**全量备份目录**。
  - 如果是**第二次增量备份**，这里填**第一次增量备份的目录**。

- **`--target-dir`**: 本次增量备份数据存放的新目录（脚本运行前该目录最好不存在，或为空）。



### **🔄 连续增量备份示例（链式）**

如果你需要进行多次增量备份（例如：全备 -> 增量1 -> 增量2），命令如下：

1. **全量备份 (基础)**
   `... --target-dir=/backup/full ...`
2. **增量备份 1 (基于全量)**
   `... --target-dir=/backup/inc1 --incremental-basedir=/backup/full ...`
3. **增量备份 2 (基于增量1)**
   `... --target-dir=/backup/inc2 --incremental-basedir=/backup/inc1 ...`
